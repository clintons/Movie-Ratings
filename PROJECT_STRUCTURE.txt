🎯 QUICK START:
═══════════════
1. Copy data file: F:\Claude\Movie Ratings.csv → ./data/
   (or use Movie Ratings.xlsx - both work!)
2. Run: ./start.sh
3. Open: http://localhost:5000


🔑 KEY FILES:
═════════════
START HERE:          INDEX.md
QUICK SETUP:         QUICKSTART.md

📦 TOTAL SIZE:
══════════════
Code:      ~50 KB
Docker:    ~500 MB (when built)
Posters:   ~5-10 MB per 100 movies
Database:  ~1 MB per 1000 movies


🚀 DEPLOYMENT OPTIONS:
══════════════════════
1. Local development:    ./start.sh
2. Proxmox LXC:          See PROXMOX_DEPLOYMENT.md
3. Cloud server:         docker-compose up -d
4. Raspberry Pi:         docker-compose up -d


💡 FEATURES:
════════════
✅ Minimalist design with white space
✅ Pagination (10 movies per page)
✅ Search by title or category
✅ Sort by: Date, Year, Rating, Category, Title
✅ Add new movies (web form)
✅ Edit existing movies (edit link)
✅ Auto movie poster fetching
✅ PostgreSQL database
✅ Docker containerized
✅ Footer link to blog.strombotne.com


🎨 DESIGN:
══════════
- Clean, professional, minimalist
- Plenty of white space
- Large movie poster thumbnails (200x300px)
- Responsive (works on phone, tablet, desktop)
- Title: "Movie Ratings Spreadsheet"
- Subtitle: "... an historical listing of movies and TV mini-series viewings rated"


🔧 TECH STACK:
══════════════
Backend:    Flask (Python)
Database:   PostgreSQL 15
Frontend:   HTML, CSS (no JavaScript frameworks)
Container:  Docker + Docker Compose
API:        OMDB (for movie posters)
